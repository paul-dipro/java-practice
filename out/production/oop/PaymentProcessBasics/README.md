# Polymorphism in Java

This folder demonstrates runtime polymorphism using an abstract class.

Concepts covered:
- Method overriding
- Dynamic method dispatch
- Abstraction using abstract class

Example:
Payment → UPI, Card, NetBanking

Key idea:
A parent reference (Payment) can call different implementations depending on the object type at runtime.

Why it matters:
This pattern is used in scalable backend systems to avoid if-else chains and make code extensible.
